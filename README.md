# Payment Flow Risk Matrix

A practical risk matrix for payment systems, wallets, fees, limits, balances, currencies, transaction states and user-facing money-flow edge cases.

## Scope

- Fiat and crypto payment flows
- Wallet states and balances
- Fee services and commission logic
- Limits and authorization rules
- Failed, pending, reversed and retried transactions
- Reconciliation and reporting
- User-facing transaction states

## Why it matters

Payment systems fail in edge states: partial failures, retries, stale balances, incorrect fees, rounding, currency conversion, provider callbacks and mismatched user-facing statuses.
